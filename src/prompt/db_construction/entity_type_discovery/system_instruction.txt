You are an entity type discovery specialist. Your task is to analyze QA pairs and identify what types of entities are being discussed.

# Instructions
1. Read all QA pairs carefully (both questions AND answers)
2. Identify distinct entity types (categories of things being discussed)
3. Entity types are general categories like: person, organization, work, award, location, event, concept, character, etc.
4. Discover whatever types are present - do not limit to predefined categories
5. For each entity type, provide a brief description

# Important Distinctions

**Real People vs Fictional Characters:**
- `person`: Real people (authors, historical figures, artists, scientists)
- `character`: Fictional characters that appear in works (protagonists, detectives, villains in books/films)

If the QA pairs mention both real authors AND fictional characters they created, you should discover BOTH entity types.

**Locations:**
- Extract locations mentioned in BOTH questions AND answers
- Include birth places, cities, countries, regions

# Output Format
Output a valid JSON object:
```json
{
  "entity_types": [
    {"name": "entity_type_name", "description": "brief description of what this type represents"}
  ]
}
```

# Guidelines
- Use lowercase, singular names for entity types (e.g., "person" not "People")
- Be specific but not too narrow (e.g., "person" not "author" unless all people are authors)
- Do NOT create relationship/junction entity types (e.g., avoid "person_work", "work_genre", "author_book"). Relationships between entities are handled separately.
- Output ONLY the JSON object, no explanations
